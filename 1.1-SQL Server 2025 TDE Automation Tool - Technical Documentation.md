### Connection String Specification

```python
Connection Format:
  DRIVER={ODBC Driver 18 for SQL Server};
  SERVER=<server_name>;
  DATABASE=<database>;
  UID=<username>;
  PWD=<password>;
  Encrypt=no;
  TrustServerCertificate=yes;
  Connection Timeout=30;

Parameters:
  - DRIVER: Must be "ODBC Driver 18 for SQL Server"
  - SERVER: Instance name or IP address
  - DATABASE: Target database (usually "master" for TDE operations)
  - UID: SQL Server authentication user
  - PWD: SQL Server user password
  - Encrypt: Set to "no" for local connections
  - TrustServerCertificate: Set to "yes" for test environments
  - Connection Timeout: 30 seconds default
  - autocommit: True (critical for BACKUP/ALTER DATABASE)
```

## Database Schema & Objects

### Master Key
```sql
-- Definition
CREATE MASTER KEY ENCRYPTION BY PASSWORD = '<password>'

-- Properties
  - Location: master database
  - Type: Symmetric key (ID = 101)
  - Purpose: Encrypts certificates and DEK
  - Access: sysadmin role only
  - Recovery: Stored in encrypted form

-- Verification Query
SELECT name, principal_id, create_date 
FROM sys.symmetric_keys 
WHERE symmetric_key_id = 101
```

### Server Certificate
```sql
-- Definition
CREATE CERTIFICATE [SonyDB_TDE_Cert]
WITH SUBJECT = 'SQL Server 2025 TDE Certificate'

-- Properties
  - Location: master database
  - Type: Asymmetric key (X.509 certificate)
  - Algorithm: RSA
  - Key Size: 2048-bit (default)
  - Issuer: SQL Server
  - Validity: 1 year (default)

-- Backup Format
BACKUP CERTIFICATE [CertName]
  TO FILE = 'path\cert.cer'
  WITH PRIVATE KEY (
    FILE = 'path\key.pvk',
    ENCRYPTION BY PASSWORD = '<password>'
  )

-- Verification Query
SELECT name, thumbprint, expiry_date, subject
FROM sys.certificates
WHERE name = 'SonyDB_TDE_Cert'
```

### Database Encryption Key (DEK)
```sql
-- Definition
USE [DatabaseName]
GO
CREATE DATABASE ENCRYPTION KEY
  WITH ALGORITHM = AES_256
  ENCRYPTION BY SERVER CERTIFICATE [CertName]

-- Properties
  - Location: User database
  - Algorithm: AES-256 (256-bit encryption)
  - Encryption: Protected by server certificate
  - Scope: Entire database (.mdf, .ndf, .ldf)
  - Re-encryption: On-the-fly during I/O

-- Verification Query
SELECT database_id, encryptor_thumbprint, key_algorithm
FROM sys.dm_database_encryption_keys
WHERE database_id = DB_ID('SonyDB')
```

### Encryption Status
```sql
-- Dynamic Management View
sys.dm_database_encryption_keys

Columns:
  - database_id: Internal database identifier
  - encryption_state: 0=No key, 1=Unencrypted, 2=In Progress, 
                      3=Encrypted, 4=Key Change, 5=Decryption, 6=Protection Change
  - encryption_state_desc: Human-readable state
  - percent_complete: Encryption progress (0-100%)
  - key_algorithm: AES_256
  - key_size: 256 (bits)
  - encryptor_thumbprint: Certificate thumbprint

Example:
  encryption_state = 3 means "ENCRYPTED"
  percent_complete = 0 means "Complete" or "At rest"
```

## API Reference

### SQLServerConnection Class

#### Methods

```python
class SQLServerConnection:
    """
    Manages SQL Server connections with automatic transaction handling.
    
    Key Features:
      - Autocommit mode enabled (critical for BACKUP/ALTER DATABASE)
      - Automatic error handling and rollback
      - Connection state tracking
    """
    
    def __init__(self):
        """Initialize connection object with None values"""
        self.connection = None
        self.cursor = None
        self.is_connected = False
    
    def connect(server, database, username, password) -> tuple:
        """
        Connect to SQL Server instance.
        
        Args:
            server (str): SQL Server instance name or IP
            database (str): Database name (e.g., 'master')
            username (str): SQL authentication username
            password (str): SQL authentication password
        
        Returns:
            tuple: (success: bool, message: str)
                - On success: (True, "Connected successfully")
                - On failure: (False, error_message)
        
        Details:
            - Opens ODBC connection with autocommit=True
            - Critical for BACKUP DATABASE and ALTER DATABASE statements
            - Stores connection and cursor objects
            - Sets is_connected flag
        
        Example:
            >>> conn = SQLServerConnection()
            >>> success, msg = conn.connect('PMSQLDBA', 'master', 'sa', 'password')
            >>> if success:
            >>>     print("Connected!")
        """
    
    def execute_query(query: str) -> tuple:
        """
        Execute SQL query without returning results.
        
        Args:
            query (str): SQL query to execute
        
        Returns:
            tuple: (success: bool, message: str)
        
        Details:
            - Executes query in autocommit mode
            - Automatically commits on success
            - Rolls back on error
            - Use for: INSERT, UPDATE, DELETE, BACKUP, ALTER
        
        Example:
            >>> success, msg = conn.execute_query(
            ...     "BACKUP DATABASE [SonyDB] TO DISK = 'path\\file.bak'"
            ... )
        """
    
    def fetch_query(query: str) -> tuple:
        """
        Execute SQL query and return results.
        
        Args:
            query (str): SELECT query
        
        Returns:
            tuple: (success: bool, results: list of tuples)
                - results[0]: First row as tuple
                - results[0][0]: First column of first row
        
        Details:
            - Executes query
            - Fetches all results into memory
            - Use for: SELECT queries
        
        Example:
            >>> success, results = conn.fetch_query(
            ...     "SELECT name FROM sys.certificates"
            ... )
            >>> if success and results:
            >>>     for row in results:
            >>>         print(row[0])  # Certificate name
        """
    
    def disconnect() -> None:
        """
        Close SQL Server connection.
        
        Details:
            - Closes connection object
            - Releases resources
            - Sets is_connected to False
        
        Example:
            >>> conn.disconnect()
        """
```

### TDEWorker Class

```python
class TDEWorker(QThread):
    """
    Worker thread for executing TDE operations without blocking UI.
    
    Key Features:
      - Runs long operations in background thread
      - Emits progress signals for UI updates
      - Thread-safe signal emission
      - Automatic error handling
    """
    
    # Signals
    progress = pyqtSignal(str)  # Emitted for progress updates
    finished = pyqtSignal(bool, str)  # Emitted on completion
    
    def __init__(operation: str, sql_conn: SQLServerConnection, 
                 parameters: dict):
        """
        Initialize worker thread.
        
        Args:
            operation (str): Operation type (backup, create_master_key, etc.)
            sql_conn: Connected SQLServerConnection instance
            parameters (dict): Operation parameters
        
        Parameters Dict:
            {
                'database': 'SonyDB',
                'certificate_name': 'SonyDB_TDE_Cert',
                'master_key_password': '...',
                'backup_path': 'C:\\...\\file.bak',
                'cert_backup_path': 'C:\\...\\cert.cer',
                'key_backup_path': 'C:\\...\\key.pvk',
                'key_password': '...',
            }
        """
    
    def run() -> None:
        """
        Execute operation in thread (called by thread.start()).
        
        Signal Emissions:
            - progress.emit(message): Every step emits timestamp + message
            - finished.emit(success, message): Final result
        
        Operations:
            - backup: Creates copy-only full backup
            - create_master_key: Creates DMK in master
            - create_certificate: Creates asymmetric certificate
            - backup_certificate: Backs up cert + private key
            - create_dek: Creates Database Encryption Key
            - enable_tde: Enables encryption on database
            - verify_tde: Checks current encryption status
            - monitor_encryption: Polls encryption progress
            - rollback_encryption: Disables TDE
            - renew_certificate: Creates new certificate
        """
```

### Signal Connections

```python
# Progress Signals
self.worker.progress.connect(self.on_operation_progress)
# When emitted: Updates output tab with timestamped message

# Completion Signals
self.worker.finished.connect(self.on_operation_finished)
# When emitted: Shows success/error dialog, updates status bar

# Example Signal Handler
def on_operation_progress(self, message: str):
    """
    Receives progress updates from worker thread.
    
    Args:
        message (str): Timestamped operation message
    
    Example message:
        "[2026-07-18 15:28:10] Starting copy-only backup of SonyDB..."
    """
    self.implementation_output.append(message)
    self.implementation_output.ensureCursorVisible()
```

## SQL Query Details

### Query 1: BACKUP DATABASE

```sql
BACKUP DATABASE [DatabaseName]
  TO DISK = 'C:\path\filename.bak'
  WITH COPY_ONLY,          -- Copy-only backup (doesn't break backup chain)
       COMPRESSION,         -- Compress backup file
       STATS = 5,          -- Progress every 5% of backup
       CHECKSUM,           -- Verify backup integrity
       FORMAT              -- Overwrite existing backup

Purpose: Create full backup before enabling TDE
Notes:
  - COPY_ONLY doesn't reset backup chain
  - COMPRESSION reduces file size ~50%
  - CHECKSUM ensures backup integrity
  - Must NOT run in explicit transaction
  - With autocommit=True: Executes successfully
```

### Query 2: CREATE MASTER KEY

```sql
USE [master]
GO

IF NOT EXISTS (SELECT 1 FROM sys.symmetric_keys 
              WHERE symmetric_key_id = 101)
BEGIN
    CREATE MASTER KEY ENCRYPTION BY PASSWORD = 'P@ssw0rd123'
END

Purpose: Create encryption hierarchy root
Notes:
  - Only one master key per database
  - Stored in encrypted form
  - Cannot be extracted
  - Required for all certificates and DEKs
  - Check: symmetric_key_id = 101 (constant)
```

### Query 3: CREATE CERTIFICATE

```sql
USE [master]
GO

IF NOT EXISTS (SELECT 1 FROM sys.certificates 
              WHERE name = 'SonyDB_TDE_Cert')
BEGIN
    CREATE CERTIFICATE [SonyDB_TDE_Cert]
      WITH SUBJECT = 'SQL Server 2025 TDE Certificate'
END

Purpose: Create asymmetric key pair for DEK protection
Notes:
  - Stored in master database
  - Subject is descriptive only
  - 2048-bit RSA key (default)
  - Valid for 1 year (default)
  - Required for database encryption
```

### Query 4: BACKUP CERTIFICATE

```sql
BACKUP CERTIFICATE [SonyDB_TDE_Cert]
  TO FILE = 'C:\path\cert.cer'
  WITH PRIVATE KEY (
    FILE = 'C:\path\key.pvk',
    ENCRYPTION BY PASSWORD = 'KeyP@ssw0rd123'
  )

Purpose: Export certificate for disaster recovery
Notes:
  - Creates two files:
    * .cer file (public certificate)
    * .pvk file (private key)
  - Private key encrypted with password
  - CRITICAL for backup restoration
  - Store offline/securely
```

### Query 5: CREATE DATABASE ENCRYPTION KEY

```sql
USE [SonyDB]
GO

IF NOT EXISTS (SELECT 1 FROM sys.dm_database_encryption_keys 
              WHERE database_id = DB_ID('SonyDB'))
BEGIN
    CREATE DATABASE ENCRYPTION KEY
      WITH ALGORITHM = AES_256
      ENCRYPTION BY SERVER CERTIFICATE [SonyDB_TDE_Cert]
END

Purpose: Create database-level encryption key
Notes:
  - Runs in database context
  - AES-256 algorithm (unbreakable)
  - Protected by server certificate
  - Enables TDE for database
  - One per database
```

### Query 6: ENABLE TDE

```sql
USE [master]
GO

ALTER DATABASE [SonyDB] SET ENCRYPTION ON

Purpose: Activate encryption on database
Notes:
  - Must run in master context
  - DEK must exist first
  - Encryption starts immediately
  - Background process encrypts pages
  - Cannot run in explicit transaction
```

### Query 7: VERIFY ENCRYPTION STATUS

```sql
SELECT
    DB_NAME(database_id) AS DatabaseName,
    CASE encryption_state
        WHEN 0 THEN 'No Database Encryption Key'
        WHEN 1 THEN 'Unencrypted'
        WHEN 2 THEN 'Encryption In Progress'
        WHEN 3 THEN 'Encrypted'
        WHEN 4 THEN 'Key Change In Progress'
        WHEN 5 THEN 'Decryption In Progress'
        WHEN 6 THEN 'Protection Change In Progress'
    END AS EncryptionStatus,
    percent_complete AS ProgressPercentage
FROM sys.dm_database_encryption_keys
WHERE database_id = DB_ID('SonyDB')

Expected Output:
  DatabaseName | EncryptionStatus | ProgressPercentage
  SonyDB       | Encrypted        | 0
```

## Error Handling

### Connection Errors

```python
Error Type: pyodbc.Error
Message: "Login failed for user 'sa'"

Causes:
  1. Wrong username or password
  2. Authentication mode not enabled
  3. User account disabled
  4. SQL Server not running

Solution:
  - Verify credentials
  - Check authentication mode (Mixed)
  - Restart SQL Server service
  - Verify account status
```

### Transaction Errors

```python
Error: "Cannot perform a backup or restore operation within a transaction"
Code: (3021)

Cause: BACKUP DATABASE executed in explicit transaction

Solution: Use autocommit=True (implemented in v2.3)
Code:     self.connection = pyodbc.connect(conn_str, autocommit=True)
```

### Path Errors

```python
Error: "Cannot open backup device 'C:\path\file.bak'"
Code: (3201)

Causes:
  1. Directory doesn't exist
  2. File permissions denied
  3. Path contains special characters
  4. Disk is full

Solution:
  - Create directory first: os.makedirs(backup_dir, exist_ok=True)
  - Verify permissions
  - Escape backslashes: path.replace('\\', '\\\\')
  - Check disk space
```

### Certificate Errors

```python
Error: "Certificate already exists"
Cause: Certificate with same name exists

Solution:
  - Use different name
  - Drop existing: DROP CERTIFICATE [CertName]
  - Rename: sp_rename

Error: "Cannot find certificate"
Cause: Certificate doesn't exist

Solution:
  - Verify certificate name
  - Check sys.certificates
  - Run CREATE CERTIFICATE first
```

## Security Considerations

### Key Management Best Practices

```
1. Master Key Protection
   ✓ Use strong password (20+ characters)
   ✓ Store password in secure vault
   ✓ Limit access to sysadmin role
   ✓ Never share password

2. Certificate Backup
   ✓ Back up .cer and .pvk files
   ✓ Encrypt backup files
   ✓ Store offline/secure location
   ✓ Document backup location
   ✓ Test restoration procedure

3. Private Key Security
   ✓ Encrypt with strong password
   ✓ Restrict file access permissions
   ✓ Use NTFS, not FAT32
   ✓ Store on encrypted drive
   ✓ Regular access audits

4. Password Management
   ✓ Master Key: 20+ chars, complex
   ✓ Key Password: 20+ chars, complex
   ✓ Change annually
   ✓ Use password manager
   ✓ Audit password access
```

### Backup Security

```
Pre-TDE Backup:
  - Unencrypted
  - Keep for rollback
  - Require access control
  - Store securely

Post-TDE Backup:
  - Automatically encrypted
  - Requires certificate keys to restore
  - Still use access control
  - Geographic redundancy recommended
```

## Troubleshooting Guide

### Encryption Not Starting

```
Symptom: Status shows "UNENCRYPTED"
Query:   SELECT encryption_state_desc FROM sys.dm_database_encryption_keys

Causes & Solutions:
1. DEK doesn't exist
   Solution: Run "CREATE DATABASE ENCRYPTION KEY" first

2. Certificate missing
   Solution: Run "CREATE CERTIFICATE" first

3. TDE not enabled
   Solution: Run "ALTER DATABASE SET ENCRYPTION ON"

4. Insufficient permissions
   Solution: Run as sysadmin role
```

### Certificate Issues

```
Symptom: "Certificate not found"

Diagnosis:
  1. Check if exists: SELECT * FROM sys.certificates WHERE name = 'CertName'
  2. Check database: Certificates stored in master only
  3. Check permissions: Requires VIEW DEFINITION

Solutions:
  1. Recreate: DROP CERTIFICATE [CertName]; CREATE CERTIFICATE...
  2. Restore: RESTORE CERTIFICATE FROM FILE
  3. Check master DB: USE [master] first
```

### Backup Failures

```
Symptom: "Backup failed"

Error: "Cannot perform backup within transaction"
Cause: autocommit not enabled
Fix:   Use connection string with autocommit=True

Error: "Cannot open backup device"
Cause: Path doesn't exist or permissions denied
Fix:   Create directory: os.makedirs(path, exist_ok=True)

Error: "Disk full"
Cause: Insufficient disk space
Fix:   Free disk space or use different path
```

### Performance Issues

```
Symptom: Slow queries after TDE enabled

Investigation:
  1. Check encryption progress
  2. Monitor CPU/disk usage
  3. Check for blocking
  4. Verify index statistics

Solutions:
  1. Run during off-peak hours
  2. Increase I/O resources
  3. Check sys.dm_exec_requests for blocking
  4. Update statistics: UPDATE STATISTICS
```

## Appendix: SQL Query Templates

### Template 1: Complete Encryption Audit

```sql
-- Run monthly for compliance audit
SELECT
    DB_NAME(k.database_id) AS DatabaseName,
    CASE k.encryption_state
        WHEN 3 THEN 'ENCRYPTED ✓'
        WHEN 1 THEN 'UNENCRYPTED ✗'
        ELSE 'OTHER'
    END AS EncryptionStatus,
    k.percent_complete AS ProgressPercentage,
    k.key_algorithm AS Algorithm,
    k.key_size AS KeyBits,
    c.name AS CertificateName,
    c.expiry_date AS CertificateExpiry,
    DATEDIFF(DAY, GETDATE(), c.expiry_date) AS DaysToExpiry,
    CASE 
        WHEN DATEDIFF(DAY, GETDATE(), c.expiry_date) < 30 THEN '⚠️ URGENT'
        WHEN DATEDIFF(DAY, GETDATE(), c.expiry_date) < 90 THEN '⚠️ WARNING'
        ELSE '✓ OK'
    END AS ExpiryStatus
FROM sys.dm_database_encryption_keys k
LEFT JOIN sys.certificates c ON k.encryptor_thumbprint = c.thumbprint
WHERE k.database_id = DB_ID('SonyDB')
```
