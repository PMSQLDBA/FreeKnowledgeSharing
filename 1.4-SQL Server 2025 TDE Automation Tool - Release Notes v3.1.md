## 📋 Detailed Changes

### Authentication System

**Before (v3.0)**:
```
- Only SQL Server Authentication
- Username field always required
- Password field always required
```

**After (v3.1)**:
```
- Dropdown: Select authentication type
- Windows Authentication: Trusted Connection
- SQL Server Authentication: Username/Password
- Fields auto-disable when not needed
- Settings saved for next session
```

### Connection String

**Windows Authentication**:
```python
conn_str = (
    f"DRIVER={{ODBC Driver 18 for SQL Server}};"
    f"SERVER={server};"
    f"DATABASE={database};"
    "Trusted_Connection=yes;"
    "Encrypt=no;"
    "TrustServerCertificate=yes;"
    "Connection Timeout=30;"
)
```

**SQL Server Authentication**:
```python
conn_str = (
    f"DRIVER={{ODBC Driver 18 for SQL Server}};"
    f"SERVER={server};"
    f"DATABASE={database};"
    f"UID={username};"
    f"PWD={password};"
    "Encrypt=no;"
    "TrustServerCertificate=yes;"
    "Connection Timeout=30;"
)
```

## 🔐 Security Notes

### Windows Authentication
- ✅ Uses current logged-in user credentials
- ✅ Password never transmitted (uses Trusted_Connection)
- ✅ More secure for domain environments
- ✅ Recommended for internal networks

### SQL Server Authentication
- ✅ Uses explicit username/password
- ✅ Works across networks
- ✅ Good for service accounts
- ✅ Works when Windows auth not available
