## 8-Phase Workflow

### Phase 1️⃣: Backup Database
Creates recovery point before encryption
- Time: 5m - 2h (depends on size)
- Output: Full backup file with compression

### Phase 2️⃣: Create Master Key
Creates encryption root key
- Time: < 1m
- Output: Master key in master database

### Phase 3️⃣: Create Certificate
Creates asymmetric key pair
- Time: < 1m
- Output: X.509 certificate (2048-bit RSA)

### Phase 4️⃣: Backup Certificate
Exports cert for disaster recovery
- Time: < 1m
- Output: .cer and encrypted .pvk files

### Phase 5️⃣: Create DEK
Creates database encryption key
- Time: < 1m
- Output: AES-256 encryption key

### Phase 6️⃣: Enable TDE
Activates encryption
- Time: 1-10m
- Output: Encryption starts transparently

### Phase 7️⃣: Verify Status
Confirms encryption status
- Time: < 1m
- Output: Shows ENCRYPTED status

### Phase 8️⃣: Monitor Progress
Tracks encryption completion
- Time: 50s
- Output: Progress percentage updates

## Troubleshooting

### Connection Issues
**Error: "Login failed"**
- ✓ Verify credentials
- ✓ Check SQL Server Authentication enabled
- ✓ Verify account not disabled

**Error: "Cannot connect"**
- ✓ Check SQL Server running
- ✓ Verify network connectivity
- ✓ Check firewall port 1433

### Operation Errors
**Error: "Backup device"**
- ✓ Create directory: mkdir C:\Backups
- ✓ Check permissions
- ✓ Verify disk space

**Error: "Certificate exists"**
- ✓ Use different certificate name
- ✓ Or drop existing certificate

### Encryption Slow
- ✓ Run during off-peak hours
- ✓ Check database size
- ✓ Increase I/O resources

## Security Best Practices

### Passwords
```
✓ Minimum 20 characters
✓ Mix of uppercase, lowercase, numbers, symbols
✓ Example: P@ssw0rd2025!Secure
✓ Store in secure vault (1Password, KeePass)
✓ Never share or log
```

### Certificate Backup
```
✓ Store .cer and .pvk files securely
✓ Encrypt backup files (BitLocker, VeraCrypt)
✓ Store offline (USB, safe deposit)
✓ Maintain 3-2-1 backup strategy
✗ Don't store with database backups
```

### Access Control
```
✓ Restrict to sysadmin role
✓ Audit all certificate access
✓ Monitor certificate usage
✓ Alert on certificate expiry
```

## Appendix: Key SQL Queries

### Check TDE Status
```sql
SELECT DB_NAME(database_id), encryption_state_desc, percent_complete
FROM sys.dm_database_encryption_keys
```

### View Certificates
```sql
SELECT name, expiry_date FROM sys.certificates WHERE name LIKE '%TDE%'
```

### Monitor Progress
```sql
SELECT TOP 1 DB_NAME(database_id), encryption_state_desc, percent_complete
FROM sys.dm_database_encryption_keys
ORDER BY percent_complete DESC
```
